## 任务名称：VoiVerse “AI 社区拟人运营”设计与原型实现

### 1. 任务背景

VoiVerse 是一个 AI-Agent 分发入口，我们的社群用户主要是对 AI 高度感兴趣、但面临信息过载的 Web3 玩家或极客。我们需要一个 **“AI 社群领航员”**，它不仅要能回答问题，还要能主动通过“信息降噪”为用户提供价值，并引导用户试用 VoiVerse 平台上的 Agent。

### 2. 核心任务目标

利用 AI 工具，为 VoiVerse 设计并实现一个能够处理社群复杂场景的“AI 运营助手”。

### 3. 三个具体的执行模块

#### **模块一：人设定义与 Prompt 工程 (策略层)**

* **任务描述：** 结合 VoiVerse 的品牌定位，为社群机器人设计一个“人设（Persona）”。
* **要求：** 提交一份结构化的提示词（Prompt）。这个机器人不能是机械的问答库，必须具备：
* **专业度：** 能听懂 Web3 和 AI 领域的专业术语。
* **性格特色：** 幽默、高情商或极客范（由候选人自行定义）。
* **降噪能力：** 按设定的频率（可配置）对群讨论内容进行简化总结，让用户不用 爬楼”。


#### **模块二：复杂运营场景模拟 (实操层)**

请针对以下两个场景，提供机器人对应的回复内容及背后的**调优逻辑**：

* **场景 A：新用户入群。** 用户进群后表现迷茫，询问“这里是干嘛的？”。机器人如何通过一句话勾起用户兴趣，并引导其去 VoiVerse 平台体验第一个 Agent？
* **场景 B：冲突化解。** 社群内有用户抱怨：“现在的 AI 资讯全是垃圾，VoiVerse 也没什么用。” 机器人如何利用“高情商”和“专业事实”化解尴尬，并将其转化为一次产品功能的科普？

#### **模块三：零代码工具部署 (技术层 - 考察上手能力)**

* **任务描述：** 使用 **Coze (扣子)** 或 **Dify**（这两个工具是目前 AI 应用工程师/运营的标配，无需代码，操作简单）搭建一个简单的 Bot 原型。
* **要求：** 1.  配置一个简单的 **知识库**（我们会提供 3-5 篇关于 VoiVerse 的介绍文档）。
2.  设置一个简单的 **工作流**：用户输入一段长文 -> 机器人调用 AI 进行降噪总结 -> 以特定格式输出到模拟对话框。
3.  **交付物：** 分享该 Bot 的体验链接或录制一段 1 分钟的运行演示视频。

---

### 4. 任务交付标准 (Success Definition)

1. 
**策略文档：** 包含机器人的人设说明、核心提示词（Prompt）逻辑 。


2. **模拟对话：** 针对上述两个场景的对话截图或文字记录。
3. 
**运行原型：** 一个可在线测试的机器人链接并操作录屏 。


4. 
**复盘思考：** 简述在调优过程中，你发现了 AI 处理 Web3 降噪信息时的哪些难点，你是如何解决的？
